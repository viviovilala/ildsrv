# ILDIS install.sh: Traefik + SSL + Superadmin Setup

**Date:** 2025-05-26  
**Scope:** Enhance `install.sh` with Traefik reverse proxy, flexible SSL configuration, nginx config externalization, and superadmin creation during setup.

---

## 1. Overview

The current `install.sh` exposes the app container directly on a user-specified port with no TLS support. This design adds:

- **Traefik reverse proxy** with smart toggle — only deployed when ILDIS is not behind an existing reverse proxy
- **Flexible SSL** — Let's Encrypt (auto), manual certificate, or HTTP-only
- **Superadmin creation** during install — replaces hard-coded admin credentials
- **Externalized nginx config** — generated by install.sh, mounted as volume, containing all security headers
- **Logging** — enable Yii2 app logs, nginx access/error logs, and Traefik access logs, persisted to volumes for easier tracing and debugging
- **Clear post-install information** — URLs, credentials, next steps

### Approach Chosen: Smart Traefik Toggle (Pendekatan 1)

The wizard asks: "Is ILDIS behind a reverse proxy?"

- **YES** → No Traefik in compose. App exposes port directly. SSL is the reverse proxy's responsibility.
- **NO** → Traefik is added to compose. User chooses SSL mode: Let's Encrypt, Manual cert, or HTTP-only.

This avoids unnecessary Traefik overhead when an external reverse proxy already handles TLS.

## 2. Wizard Flow

```
1. Installation directory           (existing)
2. Application port                 (existing)
3. Behind reverse proxy?            (NEW — YES/NO)
   ├─ YES → Skip Traefik, ask PUBLIC_DOMAIN, continue
   └─ NO  → Continue to step 4
4. SSL mode?                        (NEW — only if not behind reverse proxy)
   ├─ Let's Encrypt → Ask domain + email
   ├─ Manual cert   → Ask domain + cert path + key path
   └─ No SSL (HTTP) → Ask domain (optional, default localhost)
5. Public domain                    (adjusted based on steps 3-4)
6. Database configuration           (existing)
7. Superadmin username + password   (NEW — replaces hard-coded admin)
8. reCAPTCHA                        (existing)
9. Summary + confirm                (expanded with proxy/SSL/superadmin info)
```

New environment variables for `--non-interactive` mode:
- `BEHIND_REVERSE_PROXY` — `true` or `false` (default: `false`)
- `SSL_MODE` — `letsencrypt`, `manual`, or `none` (default: `none`)
- `SSL_DOMAIN` — domain for Traefik routing and cert
- `SSL_EMAIL` — email for Let's Encrypt (required if `SSL_MODE=letsencrypt`)
- `SSL_CERT_PATH` — path to cert file relative to INSTALL_DIR (default: `ssl/server.crt`)
- `SSL_KEY_PATH` — path to key file relative to INSTALL_DIR (default: `ssl/server.key`)
- `ADMIN_USERNAME` — superadmin username (default: `admin`)
- `ADMIN_PASSWORD` — superadmin password (required)

## 3. Docker Compose Generation

### Path A: Behind Reverse Proxy (no Traefik)

Same as current compose. App container exposes `${PORT}:80` directly. No Traefik service.

### Path B: Standalone with Traefik

Traefik service added to compose. App container does NOT expose ports directly — uses Docker labels for Traefik routing.

```yaml
services:
  traefik:
    image: traefik:v3.0
    container_name: ildis_traefik
    restart: unless-stopped
    command:
      - "--api.insecure=false"
      - "--providers.docker=true"
      - "--providers.docker.exposedbydefault=false"
      - "--providers.file.filename=/etc/traefik/config/dynamic.yml"
      - "--entrypoints.web.address=:80"
      # + entrypoints.websecure + ACME config (if SSL mode = letsencrypt)
      # + manual TLS config via dynamic.yml (if SSL mode = manual)
    ports:
      - "80:80"
      # - "443:443"  (only if SSL mode != none)
    volumes:
      - ./traefik/traefik.yml:/etc/traefik/traefik.yml:ro
      - ./traefik/config.yml:/etc/traefik/config/dynamic.yml:ro
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./ssl:/etc/traefik/certs:ro           # (if manual cert)
      - traefik_acme:/etc/traefik/acme         # (if Let's Encrypt)
      - ./logs/traefik:/var/log/traefik        # Traefik access logs
    networks:
      - ildis_net

  app:
    # NO ports section
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.ildis.rule=Host(`${DOMAIN}`)"
      - "traefik.http.routers.ildis.entrypoints=web"
      - "traefik.http.services.ildis.loadbalancer.server.port=80"
      # + TLS labels (if SSL)
    networks:
      - ildis_net

networks:
  ildis_net:

volumes:
  traefik_acme:   # (if Let's Encrypt)
```

### SSL Mode Details

| Mode | Traefik Config | Generated Files | Ports |
|------|---------------|-----------------|-------|
| Let's Encrypt | ACME resolver with TLS challenge, auto-provision | `traefik/traefik.yml`, `traefik/config.yml`, acme volume | 80, 443 |
| Manual cert | Dynamic config pointing to `/etc/traefik/certs/`, volume-mounted from `${INSTALL_DIR}/ssl/` | `traefik/traefik.yml`, `traefik/config.yml`, `${INSTALL_DIR}/ssl/` dir | 80, 443 |
| No SSL (HTTP) | Web entrypoint only, no TLS | `traefik/traefik.yml`, `traefik/config.yml` | 80 |

### Generated Files

`install.sh` generates these files in `${INSTALL_DIR}/`:

```
traefik/
  traefik.yml          — static Traefik config
  config.yml           — dynamic config (TLS, certresolver, middleware)
nginx/
  default.conf         — nginx config with security headers (externalized from image)
ssl/                   — directory for manual cert files (created empty if manual mode)
.env                   — environment variables (extended with new vars)
docker-compose.yml     — compose file (path A or B)
```

## 4. Superadmin Creation

### Removal of Default Users

The `seed_data.sql` file in `console/migrations/seed_data.sql` currently inserts 3 user rows (`admin`, `indar`, `sample`). These will be **removed** from the seed data. Only RBAC data (roles, permissions, auth_item, auth_item_child, auth_assignment for non-user entries) will remain.

### Creation During Install

After database migration succeeds, the install script creates the superadmin using the existing console command:

```bash
run_compose exec -T app php /var/www/yii user/create \
    --username="${ADMIN_USERNAME}" \
    --password="${ADMIN_PASSWORD}" \
    --role=superadmin
```

**Note:** The `user/create` command in `console/controllers/UserController.php` currently uses interactive prompts. A `--non-interactive` mode with `--username`, `--password`, and `--role` flags must be added to support automated creation.

### Wizard Input

```
Superadmin:
  Nama pengguna [admin]: ________
  Kata sandi: ________          (hidden input, min 8 chars)
  Konfirmasi kata sandi: ________ (must match)
```

Validation:
- Username: non-empty
- Password: min 8 characters
- Password confirmation must match

### Non-Interactive Mode

```
ADMIN_USERNAME=admin
ADMIN_PASSWORD=<password>
```

## 5. Nginx Config Externalization

Current state: nginx config is baked into the Docker image at `/etc/nginx/http.d/default.conf`.

New state: `install.sh` generates `${INSTALL_DIR}/nginx/default.conf` and the compose file mounts it:

```yaml
volumes:
  - ./nginx/default.conf:/etc/nginx/http.d/default.conf:ro
```

### Security Headers

All security headers live in nginx config (not in Yii or Traefik). Content varies by deployment mode:

**Always present:**
```nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Permissions-Policy "geolocation=self" always;
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self' https: data:; object-src 'none';" always;
```

**If behind Traefik (proxy), add:**
```nginx
real_ip_header X-Forwarded-For;
real_ip_recursive on;
set_real_ip_from 172.16.0.0/12;
```

**If SSL active (Let's Encrypt or Manual), add:**
```nginx
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
```

**If SSL active, CSP includes:**
```
upgrade-insecure-requests
```

**If behind reverse proxy, add trust headers:**
```nginx
# Trust forwarded headers from reverse proxy
proxy_set_header X-Forwarded-Proto $http_x_forwarded_proto;
```

Users can edit `${INSTALL_DIR}/nginx/default.conf` at any time and restart the app container to apply changes.

## 6. Logging

All logs are persisted to Docker volumes under `${INSTALL_DIR}/logs/` so they survive container restarts and are easy to inspect.

### 6.1 Yii2 Application Logs

Current state: Yii2 logs to `@runtime/logs/` inside the container (FileTarget, error/warning levels). This is lost on container recreation.

New state:
- Add a named volume `app_logs` mapped to `/var/www/runtime/logs` in the app container
- Yii2 log config **is not changed** — it already writes to `@runtime/logs/` which maps to the volume
- Post-install message shows log location

### 6.2 Nginx Access/Error Logs

Current state: nginx logs to stdout/stderr inside the container (default s6-overlay behavior).

New state: in the externalized `nginx/default.conf`, add explicit log directives:
```nginx
access_log /var/log/nginx/ildis_access.log;
error_log  /var/log/nginx/ildis_error.log;
```

In compose, add a named volume or bind mount for nginx logs:
```yaml
volumes:
  - ./logs/nginx:/var/log/nginx
```

### 6.3 Traefik Access Logs

Only present when Traefik is deployed (standalone mode). Configured via `traefik/config.yml`:
```yaml
accessLog:
  filePath: "/var/log/traefik/access.log"
  bufferingSize: 100
```

In compose, add a bind mount for Traefik logs:
```yaml
traefik:
  volumes:
    - ./logs/traefik:/var/log/traefik
```

### 6.4 Log Directory Structure

After install, `${INSTALL_DIR}/logs/` contains:
```
logs/
  nginx/
    ildis_access.log
    ildis_error.log
  traefik/          (only in standalone mode)
    access.log
```

Yii2 app logs are accessible via:
```bash
docker compose exec app ls /var/www/runtime/logs/
docker compose exec app cat /var/www/runtime/logs/app.log
```

Or via the named volume `app_logs`.
